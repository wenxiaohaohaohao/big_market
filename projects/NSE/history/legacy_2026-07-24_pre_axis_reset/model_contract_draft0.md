# Model Contract

日期：2026-07-23  
状态：Draft 0

本文件用于限制模型范围。任何新增机制必须先说明它改变哪一个核心结果，
并更新本文件和 proposition ledger。

## 1. 核心主体

- 地区 `i`
- 行业 `j`
- 异质企业生产率 `z`
- 本地居民
- 单一平台或平台渠道
- 地方政府

## 2. 基本对象

- 地区禀赋：`E_i`
- 要素价格：`w_i, r_i`
- 行业要素密集度：`theta_j`
- 单位成本：`c_ij = c_j(w_i, r_i; theta_j)`
- 企业生产率：`z`
- 渠道：本地代理 `A`、平台直连 `D`
- 平台接入成本或交易成本：`tau`
- 平台费率：`phi`
- 平台兼容型组织能力：`q_i`
- 政府能力建设投入：`G_i`

正式稿不得再用一个外生的“比较优势匹配度”标量代替成本和盈利条件。
汇总指标只能在 numerical illustrations 或未来实证映射中使用。

## 3. 必须明确的时序

1. 地区禀赋和基础设施给定；
2. 要素价格和行业成本形成；
3. 政府选择政策工具或公共投入；
4. 企业观察成本与能力后选择行业、进入和渠道；
5. 生产、销售和支付发生；
6. 地方收入、消费者福利和全国福利结算。

如果政府在企业进入之后行动，必须另写政策博弈，不能混用上述时序。

## 4. 收入与福利边界

本地第一轮收入可分解为工资、当地所有者利润、本地服务收入和本地财政净收入，
但必须通过支付流逐项定义，防止重复计算。税收是政府收入和私人支出的重新分配，
不是自动增加全国资源的价值。

正式稿必须分别报告：

- local production income；
- local disposable income；
- local government objective；
- consumer welfare；
- national welfare。

`local value capture` 不是全国福利的充分统计量。

## 5. 政策比较的准入条件

只有模型明确包含以下至少一种市场失灵时，才允许比较 facilitating policy：

- 公共基础设施的非排他性或不可分性；
- 互补投入的协调失败；
- first-mover information externality；
- learning spillover；
- 无法私人占有的组织能力外溢。

若没有市场失灵，`G_i` 只是补贴的重新命名，不能推出“赋能优于保护”。

## 6. 第一版排除

- 劳动力迁移；
- 内生地区人口与城市规模；
- agglomeration externalities；
- 多平台竞争；
- 动态组织资本；
- EL 的递归地方乘数进入主模型资源约束。

## 7. 最小闭合检查

- [ ] 偏好与需求
- [ ] 生产技术与单位成本
- [ ] 企业异质性分布
- [ ] 进入成本和渠道成本
- [ ] 平台收费或接入机制
- [ ] 要素市场与产品市场
- [ ] 政府预算约束
- [ ] 均衡定义
- [ ] 地方收入会计
- [ ] 全国资源约束和福利
